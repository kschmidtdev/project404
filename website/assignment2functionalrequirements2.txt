<h2>Overview:</h2>
<p>This scenario will cover the basics of playing the game </p>

<hr>

<p><strong>Name:</strong> Name and Password </p><p><strong>Description:</strong> The user must enter a name and password when creating a profile. </p><p><strong>Actors:</strong> ALL </p><p><strong>Use Case:</strong> </p><h3></h3><p><i>Normal:</i> </p><p>1. Type:  Characters into text field </p><p>Alternative: </p><p>1. Joypad: To select characters </p><p>2. Selection: “Ok?when finished, text affirms decision with yes or no. </p><p>3. Affirmation: User selects Yes, if no, go to 1. </p><p><i>Abnormal:</i> </p><p>2 ii) Erasing: User uses backspacing or selects backspacing to remove characters. </p><p><strong>Features Used:</strong> <ul><li>Selection Cursor </li><li>Text box </li><li>Button (affirmation yes or no, and ok) </li></ul><strong>Exceptions:</strong> <ul><li>If the user does not enter a name, the screen will respond with a message to the user, indicating they should. </li></ul></p><p><strong>Actors:</strong> </p><p><i>Max:</i> Max doesn&#x27;t necessarily mind making a new profile, because he wants to know exactly how well he does in the game. As long as he can view the time he has played the game for, and the stats he has attained in doing so, profiles are always a good idea. </p><p><i>Hubert:</i> Hubert isn&#x27;t exactly sure why he is required to use a profile, but since he wants to get to the game, he is willing to get one started. At least he&#x27;ll know he isn&#x27;t spending too much time on games. </p><p>

<p><strong>Name:</strong> Load Profile </p><p><strong>Description:</strong> The user loads a previously saved profile </p><p><strong>Actors:</strong> ALL </p><p><strong>Use Case:</strong> </p><h3></h3><p><i>Normal:</i> </p><p>1. Selection: Move the cursor to “Load Profile?</p><p>2. Selection: Move cursor over player profile with user&#x27;s name </p><p>3. Text Entry: Write password in password box </p><p><i>Abnormal 1:</i> </p><p>1. Selection: Move the cursor to “Load Profile?</p><p>2. Selection: Move cursor over player profile with user&#x27;s name </p><p>3. Text Entry: Write password in password box </p><p>4. Wrong Entry: User is notified, writes again </p><p><i>Abnormal 2:</i> </p><p>1. Selection: Move the cursor to “Load Profile?</p><p>2. Selection: Move cursor over player profile with user&#x27;s name </p><p>3. Selection: Select Cancel, if the user selects incorrect profile </p><p>4. Text Entry: Write password in password box </p><p><strong>Features Used:</strong> <ul><li>Selection Cursor </li><li>Menu </li><li>Button </li><li>Profiles </li><li>Saving/Loading </li></ul></p><p><strong>Exceptions:</strong> <ul><li>Incorrect password entry, rectified by allowing the user to enter again </li><li>Incorrect selection of profile, rectified by allowing the user to “cancel?entry. </li></ul></p><p><strong>Actors:</strong> </p><p><i>Max:</i> Loading the profile is essential for tracking his latest feats. Max is keen to do so. </p><p><i>Hubert:</i> Hubert knows he has to use his profile to play the game, so loading it is a necessity. </p><p><strong>Screen Mockups:</strong> <br/>
